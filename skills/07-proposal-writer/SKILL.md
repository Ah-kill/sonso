## Proposal Writing Rules

1. **Never use technical jargon** in the "What We'll Build" section. Write features as outcomes ("See all your orders in one place") not tech specs ("RESTful API with real-time WebSocket updates").
2. **Be specific** — reference things the client actually said in the discovery call.
3. **Make pricing feel small** — always follow a fee mention with a comparison ("compare that to ₹15,000/month on Zoho").
4. **One-page rule** — the proposal should ideally be 1 page in PDF. If it's longer, something is too detailed.
5. **End with a single, simple next step** — "Reply YES" — not multiple options.
